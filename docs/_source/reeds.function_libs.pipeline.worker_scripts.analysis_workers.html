reeds.function_libs.pipeline.worker_scripts.analysis_workers.RE_EDS_soptimization_final.do(sopt_root_dir: str, pot_tresh=0, title='', out_dir: str = None, rt_convergence=100)[source]¶

This function does the final analysis of an s-optimization. It analyses the outcome of the full s-optimization iterations. Features:

  • generate s-optimization summary

  • average roundtrip time improvement efficiency

  • deviation of sampling occurence (normalized) from the optimal sampling distribution.

The analysis of the individual s-optimization steps is stored and can be used at later executions. (if you want to analyse form scratch, keep in mind to delete the generated .npy files.)

Parameters

  • sopt_root_dir (str) – directory containing all s-optimization iterations

  • pot_tresh (int, optional) – potential energy threshold, determining undersampling (default: 0)

  • title (str, optional) – title of run (default: “”)

  • out_dir (str, optional) – ananlysis output dir, default uses the same dir like root_dir (default: None)

  • rt_convergence (int, optional) – roundtrip time convergence criterium in ps(default: 10ps)

Module contents¶

analysis workers are of general use for all gromos simulation approaches to analyse the results of the run afterwards, they can be queue to a job queue. A analysis worker is usually scheduled by a scheduler directly after the simulation approach.
