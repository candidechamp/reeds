reeds.function_libs.pipeline.worker_scripts.simulation_workers.clean_up_simulation_files module¶

After simulation cleanup This module is thought to be used as a clean up script after each run, reducing the need of storage for RE-EDS simulations on Euler or any other cluster :) it mainly removes not needed temporary files and compresses long trajectory files. It should be hanged in after each simulation step.

Module contents¶

simulation workers are of general use for all gromos simulation approaches that, need to queue jobs. A simulation worker is scheduled by a scheduler.
